About evotools

A small, growing set of open-source tools that make building on the Dash Evolution chain easier to start. Testnet-first, built in the open.

Why this exists

The hardest part of Dash Platform is the beginning. You can't just sign up — an identity has to be funded before it exists, which stops a lot of people before they write a line of code.

evotools smooths out those first steps: get a funded testnet identity in a few clicks, then learn the v4 SDK from examples that are tested and ready to run.

What's here

  • Onboard — from nothing to a funded testnet identity with a .dash name, and a ready .env.
  • Playground — run the cookbook recipes live in the browser against testnet, no install.
  • Cookbook — verified, runnable @dashevo/evo-sdk v4 recipes.
  • Next up — a project scaffolder, a data explorer, and a polished username claim.

Who's building it

I'm Bas, and online I go by 247bas. I build on Dash Platform, and I kept hitting the same walls every newcomer hits, so I started building the tools I wished existed and putting them online as I go. Everything here is open source; fork it, file an issue, or build your own on top.

I'm from the Netherlands, and I believe in where Dash is headed. The project keeps building and shipping, and I want to help more people build on it too. That's the whole point of evotools: the more builders who can get started easily, the more Dash grows into a platform serious things get built on.
